Transaction Flow

See how M2P’s Digital Travel Money APIs work

  • The customer swipes the prepaid card at an ATM or a POS
  • M2P receives the transaction from the network for authorization
  • After authenticating the card details, M2P verifies if the transaction could be approved based on the currency and set of rules
  • If the transaction currency varies from the wallet currency, the system instantly does a conversion and further authorizes the transaction
  • Even if multiple currency wallets must be used to authorize a transaction, the system would automatically make the currency exchanges
  • Then, the system will approve the transaction without user intervention
  • Finally, the transaction is completed successfully
